# FUTURE-AI Principles

The FUTURE-AI framework provides six guiding principles for trustworthy and deployable AI in healthcare. Each principle page covers the associated recommendations, practical steps, and links to the lifecycle chapters where the topic is addressed in depth.

The FUTURE-AI framework was developed by an international consortium of 117 experts from 50 countries, through a structured consensus process over 24 months. It provides 30 recommendations spanning the full AI development lifecycle.

## The six principles

| Principle | Cluster of requirements |
|---|---|
| [**F**airness](fairness.md) | Fairness, diversity, inclusiveness, non-discrimination, unbiased AI, equity |
| [**U**niversality](universality.md) | Generalisation, adaptability, interoperability, applicability, universality |
| [**T**raceability](traceability.md) | Traceability, monitoring, continuous learning, auditing, accountability |
| [**U**sability](usability.md) | Human-centred AI, user engagement, usability, accessibility, efficiency |
| [**R**obustness](robustness.md) | Robustness, reliability, resilience, safety, security |
| [**E**xplainability](explainability.md) | Transparency, explainability, interpretability, understandability |

Plus [General Recommendations](general-recommendations.md) that apply across all principles.

## Using this section

This section takes a **principle-first** view of the recommendations. If you know which principle you want to address, start here.

If you are working through the AI lifecycle phase by phase, the Design, Development, Evaluation, and Deployment sections reference relevant FUTURE-AI recommendations throughout, with callout boxes linking back to this section.
